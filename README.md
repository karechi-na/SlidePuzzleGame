## 概要
学校でのチーム制作第一回目のリポジトリです。<br>
ゲームタイトル：NumberUnion

## 制作人数
4人（プランナー2人/プログラマー2人）

## 担当
プログラミング<br>
- ゲームフローに関わる処理全般
- プロジェクト内Scriptsフォルダ以下すべて

## ゲーム内容
4×4の盤面上で同じ数字のパネルを合体させ、2048を目指すパズルゲームです。<br>
パネルを一度動かすと盤面全体がスライドするため、目先の合体だけでなく、次の配置を見据えた戦略的な判断が求められます。

## アピールポイント
本作は初めてのチーム制作作品であり、ゲーム開発だけでなく、チームで一つの作品を作り上げる難しさを学んだ作品でもあります。<br><br>
私はゲームロジックの実装を中心に担当しました。<br>
4×4のグリッド管理やパネルの移動・合体処理など、2048ならではのゲームルールを実現するため、内部データと画面表示が常に一致するよう設計を意識して開発を進めました。<br>
特に、一度合体したパネルが同じターンで再度合体しないようにする処理や、移動・合体・生成の順序管理には多くの試行錯誤を重ねました。<br>
また、初めてのチーム制作ということもあり、想定以上に担当範囲が広がる場面もありました。<br>
しかし、自分が担当した部分には最後まで責任を持ち、問題が発生するたびに原因を分析し、一つひとつ解決しながら完成まで開発を進めました。<br><br>
この作品を通して、ゲーム開発では実装力だけでなく、最後までやり切る責任感や粘り強く課題を解決する姿勢が重要であることを学びました。<br>
また、複雑なゲームロジックを整理しながら設計・実装する力を身に付けることができました。
